## Guide Philosophy

These guides aim to be:

- **Practical** - Focus on real workflows you can use today
- **Tool-agnostic** - Principles that apply across different AI tools
- **Opinionated** - Clear recommendations rather than endless options
- **Evolving** - Updated as tools and practices improve

## Suggested Reading Order

1. Start with [Getting Started](getting-started.md) to understand the basics
2. Read [Philosophy](../concepts/philosophy.md) to understand the mindset
3. Dive into [Spec-Driven Development](spec-driven-development.md) for the core workflow
4. Use [Choosing Your Tools](choosing-your-tools.md) when ready to expand
